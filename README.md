# Электронная фиксация ДТП

Монолитное многостраничное веб-приложение на Django для электронной фиксации дорожно-транспортного происшествия. Система работает как цифровой аналог электронного европротокола: водители регистрируются, создают карточку ДТП, добавляют второго участника, автомобили, повреждения и фотографии, а свидетели оставляют показания по коду доступа без регистрации.

## Стек

- Python 3.12+
- Django 5.2 LTS
- PostgreSQL
- Django ORM
- Django Templates
- HTML, CSS, JavaScript
- Bootstrap 5
- python-dotenv
- Pillow

## Основная идея

Основная идея приложения - замена привычному бумажному европротоколу. Первый водитель создает карточку ДТП, пишет свой комментарий и добавляет второго зарегистрированного водителя.

Каждый водитель отдельно заполняет свои данные и нажимает «Я всё заполнил». Кнопка отправки в ГИБДД появляется только после подтверждения обоими водителями и только у водителя с ролью «Пострадавший».

После отправки карточка получает статус «Отправлено в ГИБДД», время отправки сохраняется в `submitted_at`, а действие фиксируется в журнале `SubmissionLog`.

## Быстрый запуск одной командой

Создайте и заполните `.env`, затем выполните:

```powershell
powershell -ExecutionPolicy Bypass -File .\run.ps1
```

Скрипт `run.ps1`:

- создает `venv`, если его еще нет;
- устанавливает зависимости;
- проверяет заполненность `.env`;
- применяет миграции;
- запускает сервер разработки.

Если `.env` отсутствует, скрипт создаст его из `.env.example` и попросит заполнить параметры PostgreSQL.

## Создание базы данных PostgreSQL

Пример через `psql`:

```sql
CREATE DATABASE accident_protocol;
CREATE USER accident_user WITH PASSWORD 'strong_password';
ALTER ROLE accident_user SET client_encoding TO 'utf8';
ALTER ROLE accident_user SET default_transaction_isolation TO 'read committed';
ALTER ROLE accident_user SET timezone TO 'Europe/Moscow';
GRANT ALL PRIVILEGES ON DATABASE accident_protocol TO accident_user;
```

Файл `.env` рядом с `manage.py`:

```env
SECRET_KEY=replace-with-your-secret-key
DEBUG=True
DB_NAME=accident_protocol
DB_USER=accident_user
DB_PASSWORD=strong_password
DB_HOST=localhost
DB_PORT=5432
```

Если при `makemigrations` появляется `UnicodeDecodeError`, это не ошибка моделей. Django пытается подключиться к PostgreSQL и получает сообщение сервера в неподходящей кодировке. Проверьте, что база создана, `.env` заполнен, PostgreSQL запущен, а затем выполняйте `python manage.py migrate`.

## Основные страницы

- `/` — главная страница.
- `/users/register/` — регистрация водителя с автоматическим входом.
- `/users/login/` — вход.
- `/accidents/` — личный кабинет водителя и список доступных ДТП.
- `/accidents/create/` — создание ДТП.
- `/accidents/<id>/` — детальная страница ДТП.
- `/accidents/<id>/edit/` — редактирование общих данных ДТП до отправки.
- `/accidents/<id>/delete/` — удаление ДТП до отправки.
- `/accidents/<id>/drivers/add/` — добавление второго зарегистрированного водителя.
- `/accidents/<id>/comment/` — редактирование комментария текущего водителя.
- `/accidents/<id>/ready/toggle/` — подтверждение или снятие готовности текущего водителя.
- `/accidents/<id>/vehicles/add/` — выбор автомобиля из профиля водителя.
- `/accidents/<id>/damages/add/` — добавление повреждения.
- `/accidents/<id>/photos/add/` — загрузка фотографии ДТП.
- `/accidents/<id>/submit/confirm/` — подтверждение отправки в ГИБДД.
- `/gibdd/accidents/` — список всех ДТП для сотрудника ГИБДД с сортировкой.
- `/witness/` — ввод кода свидетеля.
- `/witness/<code>/statement/` — форма свидетельского показания.

## Роли пользователей

### Водитель

Водитель регистрируется, входит в систему, создает ДТП, просматривает свои ДТП и ДТП, где он указан участником. До отправки он может добавлять свои данные, автомобили из своего профиля, повреждения, фотографии и редактировать свой комментарий. Каждый водитель отдельно подтверждает, что он все заполнил.

### Свидетель

Свидетель не регистрируется. Он открывает страницу `/witness/`, вводит код доступа, видит краткую информацию о ДТП и отправляет показания, контактный телефон и при необходимости фотографию.

### Сотрудник ГИБДД

Сотрудник ГИБДД входит по заранее созданному аккаунту, видит все ДТП на странице `/gibdd/accidents/`, сортирует список и открывает полную карточку ДТП со всеми участниками, автомобилями, повреждениями, фотографиями и свидетельскими показаниями.

## Проверка номеров

Форма автомобиля проверяет:

- госномер в формате `А123ВС77` или `А123ВС777`;
- VIN из 17 латинских букв и цифр без `I`, `O`, `Q`;
- полис ОСАГО в формате `ЕЕЕ1234567890`.

Для госномера и ОСАГО пробелы и дефисы удаляются, латинские похожие буквы приводятся к кириллице.

## Отправка в ГИБДД

Отправка доступна только если:

- добавлены ровно два водителя;
- оба водителя подтвердили готовность;
- текущий пользователь имеет роль «Пострадавший»;
- ДТП еще не было отправлено.

После подтверждения приложение:

- сохраняет дату и время отправки в `submitted_at`;
- создает запись `SubmissionLog`;
- показывает страницу успешной отправки;
- блокирует редактирование ДТП, комментариев, автомобилей, повреждений и фотографий через личный кабинет.
